2.2 觸發模式 (Trigger Mode)

UI 名稱API 常數RecoMode 值說明
On-the-flyRecognitionMode.MODE_ICR—即時辨識 — 每寫完一筆就觸發辨識，結果即時更新

2.3 RecoMode（.conf 中的辨識模式值）

值含義適用場景
0單筆即時一筆結束立刻辨識（Shape、Gesture 使用）
2多筆批次 (ECR/ICR)收集多筆後辨識（Text、Math、Diagram 使用）
10原始輸出不做美化/結構化，直接輸出座標+分類（raw-content 使用）

Math 輸出流程 — LaTeX 管線

手寫筆跡 ──→ 引擎辨識 ──→ resultPartiBean.getLabel()
                                    │ (就是 LaTeX 字串)
                ┌───────────────────┘
                ├→ CommonUtils.convertMath() → MathJax 渲染 → 顯示數學式
                ├→ mEditor.latexToMathML() → MathML 格式
                ├→ mEditor.latexToOfficeMathML() → Office MathML (Word 格式)
                └→ FileUtils.saveLatex() → 存成 .tex 檔案
IMPORTANT:
引擎辨識出的結果本身就是 LaTeX 格式。Demo 中看到的漂亮數學式是用 MathJax (WebView) 渲染 LaTeX 得到的。方向是：手寫 → LaTeX → 渲染，不是反過來。

五、與 nOCR 整合轉換

✅ Input 完全相容：nOCR 的 PointerEvent 可輕易轉為 FluxInk 的 RecognizePoint(x,y,t,action)。
🟡 Output 需轉換：FluxInk 輸出原生 JSON 包含 label、words、chars、bounding_box，這些資料皆齊備，能完美反向封裝成 nOCR 預期的 JiixDefinitions。(尤其以 Diagram JSON 結構更為明確)
